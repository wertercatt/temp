About

Adventure is a flexible engine for building “choose-your-own-adventure” type stories. An Adventure can have any number of pages and branches, to create a complex story with multiple interlinking paths, and multiple endings; or maybe as linear as you want! Adventure was inspired by watching people play “choose-your-own” styled web-forum games, and realizing how unsuitable that platform is for branching stories.

If you find something that looks like a bug, be sure to use the tracker to report it!

Quick Start

  1. Register an account if you’d like to make an adventure that only you can edit.
  2. Click the “Start New” link in the top right, give your adventure a name and description.
  3. You’ll be given an edit page. Copy/Paste URLs from Youtube, Imgur, etc. to add content. Add some text if you’d like.
  4. Create some actions to give users choices on what to do next. A new page is automatically created.
  5. Click the action you’ve created to go to the new page. Add content to that page.
  6. Repeat until you have a story!

Using Adventure

Content Editing

An Adventure consists of three item types: pages, content, and actions. Each page can have any arbitrary number of content and actions items associated with it.

Once created, Text content and actions can be edited. Other types of content should be deleted and recreated if you’d like to change them.

Content and actions can be reordered using the “up” and “down” arrows next to each when in edit mode.

Each page has a Page ID associated with it. It’s a number that can be found at the top right of each page, next to the editing actions. You can use this ID to create an action that links to it from any other page. When you create a new action, a page is automatically created and associated with it. If you want to link to an existing page, use the “Existing page” section when adding an action, and select the target page that you want to link.

Supported Content Types

Video (Youtube)

With Youtube, you can paste almost any link or video ID, and have it embed like you’d expect. They’ll look something like these:

  • http://www.youtube.com/watch?v=59cQWw9ctOA
  • http://youtu.be/59cQWw9ctOA
  • 59cQWw9ctOA
  • Or most any URL based on those…

Image (Imgur)

With Imgur, you can past any URL to an image, or the direct ID, such as these:

  • http://imgur.com/gallery/7CJ5R
  • http://i.imgur.com/7CJ5R.jpg
  • http://imgur.com/7CJ5R
  • 7CJ5R

Audio (SoundCloud)

For SoundCloud, you must paste the URL directly to the song that you’re trying to embed. Anything else might have unexpected results. The URL will look like this:

  • http://soundcloud.com/martonmedveczky/rhodes

Text

Text in Adventure is parsed with Textile, a lightweight markup language for text styling. You should be able to accomplish everything you need with it, including text styles, coloring, lists, etc. Read a good quick reference on it at http://textile.thresholdstate.com/, or the full reference at http://redcloth.org/textile.

Permissions

In Adventure, you have the option to create an account. It’s not necessary to view or create adventures, but it is if you want to lock an adventure. There are two permissions states relating to adventures. Note that the adventure’s permission state applies to all of the pages, content, and links it contains:

  • Unlocked Adventures are created by non-registered and registered users. Anyone can edit them, including other non-registered users. Exciting!
  • Locked Adventures are created by registered users, who choose to select the “lock” option when creating a new adventure. These adventures are editable by the creator only, so you can make a curated adventure that everyone can view.

Currently, all adventures are visible to everyone regardless of registration status. I don’t forsee changing that in the near future.

Editors and Collaborative Editing

If you’ve registered an account and created your Adventure, you can add Editors to it. They’ll have all the same abilities as you do on your Adventure, with the following exceptions:

  • Editors cannot add or remove additional Editors
  • Editors cannot modify the Adventure itself, such as the name, or global CSS

Suggestions

So you’re stuck? You need some input from your readers? OK! Adventure allows you to enable Suggestions on individual pages in your Adventure. Users can then send you their ideas for your next action, and you’ll receive an email letting you know a suggestion has been received.

From there, you can choose to approve or deny the suggestions you’ve received, and keep on writing!

Important Notes!

Adventures allow you to set custom per-adventure CSS. Please, please, please don’t turn this into myspace or I will disable it right quick.

Making this flexible also makes things a little convoluted at times:

  • A single page can have many actions leading to it. A page takes its name from the action taken. This means that technically a page can have a few different names/descriptions. You’ll see this best if you choose to list all pages in an adventure; Page IDs may repeat, since they are linked from multiple places.
  • In the same way, a page is deleted when all links to that page are removed! Technically, it still exists somewhere, but could be a challenge to find!

Unimportant Notes

  • Adventure only targets modern browsers, often only their current versions. I am not even marginally interested in fixing browser bugs.
  • If Adventure zones out for a bit around 3 or 4 am, it’s because some background scripts are busy cleaning things up. This is normal. Do not be alarmed.
